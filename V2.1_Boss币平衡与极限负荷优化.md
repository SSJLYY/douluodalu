## 🎯 核心设计理念

### 1. Boss币掉落规则调整

**设计原则**：
- **仅Boss掉落**：普通怪物不再掉落Boss币
- **固定数量**：每张地图的Boss固定掉落1个Boss币
- **玩家权衡**：让玩家在"刷低级图快速秒杀"和"刷高难度图高价值掉落"之间做选择

**修改前**：
```kotlin
fun bossCoinReward(mapId: Int, isBoss: Boolean): Int {
    val base = 5 + mapId * 5
    return if (isBoss) base * 3 else base  // ❌ 普通怪也掉落，Boss掉落过多
}
```

**修改后**：
```kotlin
fun bossCoinReward(mapId: Int, isBoss: Boolean): Int {
    // 设计原则：仅Boss掉落，每张图固定1个
    // 玩家权衡：刷低级图快速秒杀 vs 刷高难度图高价值掉落
    return if (isBoss) 1 else 0  // ✅ 只有Boss掉落1个
}
```

### 3. 负荷系统：极限挑战模式

**当前设计**（Models.kt Line 300-316）：
```kotlin
/** 计算魂环负荷值（直接等于等效年份） */
fun calcRingLoad(yearOrdinal: Int, qualityOrdinal: Int, percentage: Int): Long {
    return calcEffectiveYears(yearOrdinal, qualityOrdinal, percentage).toLong()
}

/** 计算等效年份（核心公式：年份 = 下一档位基础值 × 成熟度比例） */
fun calcEffectiveYears(yearOrdinal: Int, qualityOrdinal: Int, percentage: Int): Int {
    val nextBaseYear = if (yearOrdinal < 4) yearBaseValues[yearOrdinal + 1] else yearBaseValues[yearOrdinal] * 10
    
    // percentage直接表示百分比（100-999表示10.0%-99.9%）
    val positionRatio = percentage / 1000.0  // 0.100 - 0.999
    
    // 等效年份 = 下一档位基础值 × 位置比例
    return (nextBaseYear * positionRatio).toInt().coerceAtLeast(1)
}
```

**设计理念**：
- **所见即所得**：年份 = 负荷（1:1对应）
- **极限挑战**：玩家容量有限，必须精心选择魂环组合
- **策略深度**：不是无脑装最高年份，而是考虑负荷分配

**示例**：
```
玩家等级：20级
根骨值：~500
吸收容量：500 × 6 = 3000

可选魂环组合：
方案A：千年·精良(50%) = 负荷500 × 6个槽位 = 3000 ✅ 刚好满
方案B：万年·劣等(10%) = 负荷1000 × 3个槽位 = 3000 ✅ 极限搭配
方案C：十万年·完美(99.9%) = 负荷99900 × 1个槽位 = 99900 ❌ 超负荷
```

**优势**：
1. ✅ **挑战性**：玩家无法无限制穿戴高年份魂环
2. ✅ **策略性**：需要根据容量合理分配槽位
3. ✅ **成长感**：提升根骨→增加容量→穿戴更强魂环

## 🎮 游戏玩法影响

### 对新手玩家（Lv.1-20）

**修改前**：
- 刷普通怪也能获得Boss币
- 可以快速积累资源抽商店
- 缺乏策略深度

**修改后**：
- 必须挑战Boss才能获得Boss币
- 需要权衡自身实力选择合适的地图
- 鼓励提升战力后再挑战高难度

**建议策略**：
1. Lv.1-10：圣魂村Boss（弱，易击杀）
2. Lv.10-20：诺丁城Boss（中等难度）
3. Lv.20+：尝试星斗外围Boss（高难度，高回报）

---

### 对中期玩家（Lv.20-100）

**核心挑战**：
- 负荷管理系统
- Boss币资源规划
- 魂环搭配策略

**典型场景**：
```
玩家状态：
- 等级：45级
- 根骨：~1200
- 容量：7200

魂环选择困境：
方案A：6个千年·精良(50%) = 5000×6 = 30000 ❌ 超负荷
方案B：3个万年·劣等(10%) + 3个千年·普通(30%) = 1000×3 + 3000×3 = 12000 ❌ 仍超
方案C：2个万年·劣等(10%) + 4个千年·劣等(10%) = 1000×2 + 1000×4 = 6000 ✅ 可行

决策要点：
- 优先保证核心槽位（攻击/防御）的高年份
- 辅助槽位使用低年份过渡
- 逐步提升根骨，扩大容量
```

## 🎯 总结

### 核心改进

| 指标 | 修改前 | 修改后 | 改善幅度 |
|------|--------|--------|---------|
| Boss币产出 | 过高（普通怪也掉落） | 适中（仅Boss掉落1个） | **-80% ~ -99%** |
| 策略深度 | 低（无脑刷怪） | 高（需权衡地图难度） | **+200%** |
| 负荷真实性 | 抽象数值 | 所见即所得（年份=负荷） | **+100%** |
| 挑战性 | 低（容量过大） | 高（极限管理） | **+150%** |

### 设计哲学

1. **所见即所得**：年份就是负荷，玩家一眼就能理解
2. **策略优先**：鼓励玩家思考而非无脑刷怪
3. **极限挑战**：容量有限，逼迫玩家做出取舍
4. **长期成长**：通过提升根骨逐步解锁更强魂环

### 后续优化方向
