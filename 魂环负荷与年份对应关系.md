## 🎯 核心设计理念

### 负荷 = 年份数值

**设计原则**：
- **百年魂环**(100-999年) → 负荷 **100-999**
- **千年魂环**(1000-9999年) → 负荷 **1000-9999**
- **万年魂环**(10000-99999年) → 负荷 **10000-99999**

**优势**：
1. ✅ 直观易懂：负荷值就是年份，玩家一眼就能理解
2. ✅ 便于规划：容量4000 → 最多吸收千年40%的魂环
3. ✅ 平衡性好：根骨系统真正发挥作用

## 📊 计算公式详解

### 1. 负荷计算

```kotlin
fun calcRingLoad(yearOrdinal: Int, qualityOrdinal: Int, percentage: Int): Long {
    val baseYear = yearBaseValues[yearOrdinal]  // 百年=100, 千年=1000...
    
    // 品质影响：精良品质的百年魂环，负荷相当于百年档位的1.5倍
    val qualityAdjusted = baseYear × qualityMult[qualityOrdinal]
    
    // 成熟度影响：0%=基础值，100%=基础值×1.9（线性增长）
    val pctMult = 1.0 + percentage × 0.009  // 0%=1.0, 100%=1.9
    
    return qualityAdjusted × pctMult
}
```

**参数说明**：
- `yearOrdinal`: 年份档位 (0=百年, 1=千年, 2=万年, 3=十万年, 4=百万年)
- `qualityOrdinal`: 品质等级 (0=劣等, 1=普通, 2=精良, 3=优秀, 4=完美)
- `percentage`: 成熟度 (0-100%，表示在档位内的位置)
- `qualityMult`: 品质倍率 [1.0, 1.2, 1.5, 1.8, 2.2]

---

### 2. 等效年份计算

```kotlin
fun calcEffectiveYears(yearOrdinal: Int, qualityOrdinal: Int, percentage: Int): Int {
    val baseYear = yearBaseValues[yearOrdinal]
    
    // 同档位内，品质和成熟度共同决定具体年份
    val qualityOffset = baseYear × (qualityMult[qualityOrdinal] - 1.0) × 0.5
    val pctOffset = baseYear × percentage × 0.009
    
    return baseYear + qualityOffset + pctOffset
}
```

**示例**：优良·百年(79.8%)
```
baseYear = 100 (百年)
qualityMult[3] = 1.8 (优秀)
percentage = 79.8

qualityOffset = 100 × (1.8 - 1.0) × 0.5 = 40
pctOffset = 100 × 79.8 × 0.009 = 71.8

等效年份 = 100 + 40 + 71.8 ≈ 212年
```

## 🔧 技术实现细节

### 1. 数据结构

```kotlin
// Models.kt
object SoulRingSystem {
    // 年份档位基础值
    private val yearBaseValues = intArrayOf(100, 1000, 10000, 100000, 1000000)
    
    // 品质倍率
    private val qualityMult = doubleArrayOf(1.0, 1.2, 1.5, 1.8, 2.2)
    
    // 计算负荷
    fun calcRingLoad(yearOrdinal: Int, qualityOrdinal: Int, percentage: Int): Long {
        val baseYear = yearBaseValues[yearOrdinal.coerceIn(0, 4)]
        val qualityAdjusted = baseYear * qualityMult[qualityOrdinal.coerceIn(0, 4)]
        val pctMult = 1.0 + percentage * 0.009  // 0%=1.0, 100%=1.9
        return (qualityAdjusted * pctMult).toLong().coerceAtLeast(1)
    }
    
    // 计算等效年份
    fun calcEffectiveYears(yearOrdinal: Int, qualityOrdinal: Int, percentage: Int): Int {
        val baseYear = yearBaseValues[yearOrdinal.coerceIn(0, 4)]
        val qualityOffset = baseYear * (qualityMult[qualityOrdinal.coerceIn(0, 4)] - 1.0) * 0.5
        val pctOffset = baseYear * percentage * 0.009
        return (baseYear + qualityOffset + pctOffset).toInt().coerceAtLeast(1)
    }
    
    // 显示名称
    fun fullDisplayName(yearOrdinal: Int, qualityOrdinal: Int, percentage: Int): String {
        val yearName = RingYear.entries[yearOrdinal].displayName
        val qualName = RingQuality.entries[qualityOrdinal].displayName
        val effYears = calcEffectiveYears(yearOrdinal, qualityOrdinal, percentage)
        return "${qualName}·${yearName}(${percentage / 10}.${percentage % 10}%≈${effYears}年)"
    }
}
```

## 📝 总结

### 核心改进

1. ✅ **负荷=年份**：直观易懂，便于规划
2. ✅ **百分比精确**：成熟度以小数显示（79.8%）
3. ✅ **品质前置**：显示格式改为"品质·年份"
4. ✅ **容量限制清晰**：玩家能准确判断能否吸收

### 玩家体验提升

- **新手友好**：看到"容量3000"就知道能吸收千年以下的魂环
- **策略深度**：需要在品质和成熟度之间权衡
- **成长明确**：提升根骨→增加容量→吸收更高年份魂环
